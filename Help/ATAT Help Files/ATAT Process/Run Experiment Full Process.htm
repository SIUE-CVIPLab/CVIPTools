Full Experiment Process:

 

Step 1: Create Image Set to Test

This will act as a step by step tutorial of setting up and running an experiment. During this example, five test images will be used with 6 unique algorithms. The test images will be found in the “Test Images” folder and the ideal images for the test will be found in the “Ideal Images” folder. I placed these folders together in the Example Project Images folder.

The test images are a set of bone cancer images. To get the ideal images, the test images were put through an algorithm using the CVIPtools GUI. The algorithm was an Otsu segmentation into a morphological closing circle with a diameter of 3 pixels. These images were then saved into the Ideal Images folder.

Step 2: Create Project

Once the ATAT application is opened, the first step in starting a new experiment is to create the project folder. This folder will be used to store results, configuration settings, and the file used to reopen the project. To create a new project, click File->New Project. This will open the new project dialog. Give the project a name, and the location of where to store it. The browse option can be used to easily navigate your file structure. Once these are selected, click OK. This will create a new folder with the project name title inside the location specified. I decided to place the project folder in the same location as the image folders.

Step 3: Add Images

When a project is created, the first form displayed is the images section. This is where we will add the test and ideal images that were defined in step 1. To add our images, the “Add Image” button will be clicked. This will bring up a file explorer tab to find the images. We will find the Test Images folder and select the images inside. Clicking Open will add the selection. If the images were added correctly, they should now be displayed on the table. To add the ideal images, we will click the “Add Ideal Images” button. Instead of a file, the ideal images will be found by selecting a folder. This folder should hold all ideal images that will be added. Note that the ideal images must have the same name as the Test Image counterpart and should be added second. Once again, find the correct folder and click OK to add. If the table updates with rows with new images, the ideal images were added correctly.

Step 4: Define Algorithms

With the images added, we need to define the algorithm to test. In our case, we know the initial algorithm used to create the ideal images. In most applications, this will not be known and it is the ATAT programs job to determine what algorithms work best. So, we can start by testing a simple algorithm combination including color to gray transformation and segmentation methods. Lets add these to the table. The first step to adding a process is to select the process. To do this, click the drop down menu below the “Select Process” text. This will create a drop down list of all the available functions in ATAT. We will select Color-to-Gray Conversion - Average. Color-to-Gray Conversion - Average has no parameters, so we do not have to change them. The last step is to select a stage. Functions will be implemented starting with stage 1. We know the Color-to-Gray Conversion - Average function should be the first function, so it will be assigned stage 1. Click “Add Process” to add the function at that stage. This should add the function to the table.

The average is not the only stage 1 function we want to add. We want to compare the average to the luminance function. Following the process above, we add this new function to the same stage. The luminance, again, has no parameters and will once again be added to stage 1.

Binary thresholding can then be added to stage 2 as a process implemented on the output of stage 1. The only parameter required is the threshold value, which can be implemented using the first parameter. By making the lower 20 and upper 30 with an increment of 10, two binary threshold processes will be added to stage 2 with threshold values of 20 and 30. The lower bound of the parameter will be added first. Then, the parameter will increment until it reaches the upper bound. It will add the function with each parameter change

The binary threshold is not the only stage 2 function we want to add. We want to compare the process to another segmentation function, the auto threshold process. Following the process above, we add this new function to the same stage. The Auto function also has a single parameter. We want the auto function to only run once, so for the parameter the lower and upper bounds are the same with an increment of 1. Finally, we set the stage to 2 and hit “Add Process”. Because both functions are stage one, they will essentially be implemented at the same time and will not by in the same algorithms.

These functions will result in the following paths. This means there will be 6 algorithm result images per test image. Our total image count would be 5(test images) * 6(Algorithms) = 30.

Step 5: Define Test Parameters

All the items needed for the experiment are now added. We just need to define the test. There are three types of tests that revolve around two experiment sections. The first is the testing phase. This will take the test images and run them through each algorithm. A results folder will be created with the new images. The second phase is the comparison. The comparison will take the images created by the algorithms and compare them to the ideal images. This will result in an excel sheet with the error metric results. For this example, we will be using the “Full Test Option”. This will do both phases. On the test and compare section, click the “Full Test” option if it is not already selected. This test includes the comparison phase, so we must specify error metrics. There are seven error metrics used to measure the similarity between ideal and result images. If an error metric is checked, the success thresholds can be defined. These will be used to define if the measured error is successful or not. To compare algorithms, choose the average and/or standard deviation options. These will combine each unique algorithm’s result. The settings for this experiment are shown with the Dice and RMS options chosen along with the average option.

Click Run to begin the test. The test can be paused or cancelled at any time. The progress bar will fill with the number of algorithms left. The bar will also state if it is in the testing or comparison phase.

Running a Test phase experiment is required for the first run because no result images were made yet. But after the initial test phase, the “Only Compare” option can be used to bypass the algorithm implementation. This allows the test phase to be skipped and the comparison measurements to begin immediately. All that is needed is a results folder (this will be the outputFolder of a previous test most of the time) and an ideal folder. The comparison will use these two folders for its error measurement results.

Step 6: Analyze Results

The final part is to analyze the results of the test. All result images and files will be placed into the project directory. The result images will be placed in the “outputImage” directory and two excel sheets titled image results and algorithm results holding the comparison measurements, located in the project folder. The output folder will be organized by algorithm. These images can be opened with the use of the CVIPtools GUI. Also, clicking a row in the results table of ATAT will display both the result image and the ideal image.

The two excel sheet represent either the individual image performance or the algorithm performance overall. These will be sorted by column in the order of highest to lowest error metrics checked, and also by band. These can tell us which algorithm performed the best and how each individual image performed.
